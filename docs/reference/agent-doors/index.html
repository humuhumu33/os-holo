Agent doors

An agent does not need a key or an account to use Hologram OS. It starts at one self-verifying discovery document, reads the live list of doors, and acts — every result is an object it re-derives before trusting (Law L5). Don’t trust; re-derive.

Start here

Section titled “Start here”

agents.json is the single entry point. It is authoritative — read the doors from it, not from this page, because the live document changes as the OS grows.

Re-derive everything

Section titled “Re-derive everything”

Every tool result and every resource is content-addressed. Before you act on a result, re-derive its hash and compare it to the κ it claims — see the κ-route. A mismatch is refused, not warned.

Machine-readable docs

These docs ship their own agent index at /docs/llms.txt, plus the reference as data: conformance.json and adrs.json. Consume the docs as structure, not prose.
