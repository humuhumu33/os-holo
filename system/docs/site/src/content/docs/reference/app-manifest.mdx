`holospace.json` is an app's manifest: a W3C `SoftwareApplication` that declares what
the app is and what privileges it needs. Its sibling `holospace.lock.json` pins every
asset byte, and the lock's root κ is the app's identity. Generated by
[`holo-scaffold.js`](https://github.com/humuhumu33/os-holo/blob/main/system/os/usr/lib/holo/holo-scaffold.js).

## Shape

```json
{
  "id": "org.example.hello",
  "entry": "index.html",
  "icon": "icon.svg",
  "capabilities": {
    "storage": ["org.example.hello"],
    "permissions": ["camera"],
    "channels": []
  },
  "shared": ["holo-mobile.css", "holo-theme.js"]
}
```

## Fields

| Field | Type | Meaning |
| --- | --- | --- |
| `id` | string | Reverse-DNS identifier, e.g. `org.example.hello`. Names the app's storage scope. |
| `entry` | string | The entry page mounted in the sandboxed frame, e.g. `index.html`. |
| `icon` | string | The branded glyph (inherits the app's accent via `currentColor`). |
| `capabilities.storage` | string[] | Storage scopes the app may use. A non-empty list grants `allow-same-origin` in the sandbox. |
| `capabilities.permissions` | string[] | Permissions-Policy features (e.g. `camera`, `microphone`) surfaced via the frame's `allow`. |
| `capabilities.channels` | string[] | Cross-app channels; a non-empty list grants `allow-popups`. |
| `shared` | string[] | Core OS modules the app links (e.g. `holo-mobile.css`, `holo-theme.js`). |

## Capabilities become the sandbox

The launcher derives the iframe sandbox from the declared capabilities — least
privilege by construction
([`holo-launch.mjs`](https://github.com/humuhumu33/os-holo/blob/main/system/os/lib/holo-launch.mjs)):

```js
export function capabilitiesToSandbox(caps = {}) {
  const sandbox = ['allow-scripts'];
  if ((caps.storage || []).length)  sandbox.push('allow-same-origin');
  if ((caps.channels || []).length) sandbox.push('allow-popups');
  const allow = (caps.permissions || []).filter((p) => FEATURE.has(p)).join('; ');
  return { sandbox: sandbox.join(' '), allow };
}
```

An app that declares nothing runs with `allow-scripts` only. It gets exactly the
privileges it asks for, no more.

<Aside type="note" title="Identity lives in the lock">
  `holospace.json` declares the app; `holospace.lock.json` pins it. Edit any asset
  and the lock no longer re-derives — run the resealer so the app's κ tracks its
  bytes (Law L5):

  ```sh

  ```
</Aside>
