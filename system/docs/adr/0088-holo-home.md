# ADR-0088: Holo Home — the desktop becomes a content-addressed holospace: every holospace (now including the home itself) is ONE κ-object, and render · link · remix · share · provenance follow from that single fact

**Status:** Accepted (the scene-manifest model landed + Node-witnessed 11/11; the compositor refactor + Create wiring staged). The foundation ships at `system/os/usr/lib/holo/holo-home-scene.mjs` (pure, dependency-free). It makes the home/desktop a deterministic render of a small canonical **manifest** that REFERENCES the screen rather than serializing it: a wallpaper **seed** κ (Holo Cosmos, ADR-0080), each placed κ-object as `{ k, x, y, w, h, state? }`, and dock/theme config κs — sealed (RFC 8785 JCS + SHA-256) to the home κ. `render(manifest)` instantiates the desktop and `serializeWorld(...)` reads it back, and the two are **inverse** (the Holo Atlas isomorphism) — so the desktop and its κ are the same object viewed two ways. Witnessed in Node 11/11: re-derivable (same desktop → same κ, object-order-independent); the render⇄serialize round-trip returns the identical κ; provenance is metadata (lineage doesn't change the scene address); `remix(parent, edit)` yields a distinct child κ linked `prov:wasDerivedFrom` the parent (Holo Prov, ADR-0082); content convergence (undoing an edit returns to the parent κ); `walkLineage` walks child → genesis. The **desktop↔scene bridge landed** (`system/os/usr/lib/holo/holo-home.mjs`, Node-witnessed 7/7): the shell's live desktop (`desktop.doc().world` nodes + `HoloWidgets.snapshot()` + the wallpaper seed + layout) serializes to ONE `holo:HomeScene` κ via the general engine (ADR-0089) and round-trips (`worldToScene ⇄ sceneToWorld`, mixed app-nodes + widgets, same κ) — and crucially gains a **provenance chain** (remix the desktop → a child home κ `wasDerivedFrom`, with `walkLineage` to genesis) that the shell's own `desktop.kappa()` repo-κ lacked, plus per-object live STATE via the widget schema (a runtime tick → the same κ). **Deferred (the load-bearing remaining work):** re-architecting the shell's desktop boot to `render(homeκ)`, and the `csResolve` desktop case → the home κ (after which the existing Create window→preview FLIP and Done-reverse apply to the desktop with no special case). The READ half (live desktop → remixable κ with lineage) is now safe + done; only the live boot/preview-render wiring remains, staged deliberately so it doesn't break the live desktop.

**Context.** Everything on the substrate is content-addressed — an app is a single κ-document, so it already has the five capabilities that follow from a content address: it **renders** (κ → screen, re-derivably), **links** (`holo://κ`), **remixes** (edit → a child κ), **shares** (hand someone the κ; they re-derive the exact thing), and carries a **provenance chain** (ADR-0082). Verified live: opening an app into the Create studio loads its exact source, FLIPs the running window into the preview, and reverses on Done. The one hole was the **desktop itself**: it was the shell's live compositor — an animated cosmos `canvas`, composited live κ-objects, a dock — *not a document*. So it could not be mirrored, remixed, shared, or tracked; entering Create from the empty desktop scaffolded a fresh "Hello, holospace" because there was nothing to mirror. The asymmetry is the bug: an *app* is a κ-document but the *desktop* — the most-seen holospace of all — was not. Either the desktop is faked into a document (lossy DOM serialization — the cosmos, animations, and live objects won't round-trip, and the result is not one re-derivable κ) or the desktop is *genuinely made* a content-addressed object. This ADR takes the second path.

**Decision.** Make the desktop a deterministic render of a content-addressed scene manifest, so the five verbs become **properties of the κ, uniform across every holospace kind** — app, widget, experience, and home:

1. **The Home Scene manifest — references, never snapshots.** One small canonical document: `wallpaper` is a **seed κ** (the cosmos re-derives from the seed, ADR-0080 — never a pixel capture, so the manifest is tiny and lossless); `objects[]` are placed κ-objects `{ k, x, y, w, h, state? }` (a reference + layout + a *defined minimal* round-trip state, not arbitrary live DOM); `dock`/`theme` are config κs. Objects are canonicalized by `(k,x,y)` so the κ is a pure function of the *set* of placed objects + layout, independent of mount order. The manifest IS the home κ.
2. **The desktop and its κ are one object, two views (isomorphism).** `render(manifest) → desktop` and `serializeWorld(desktop) → manifest` are inverse — witnessed by the round-trip returning the identical κ. This is what makes the desktop a *real* document rather than a mirrored fiction: the live screen and the manifest cannot disagree, because each is derivable from the other.
3. **Re-derivable, shareable, with lineage.** `sceneKappa` addresses the *content* (wallpaper ⊕ objects ⊕ dock ⊕ theme), so the same desktop re-derives to the same κ on every device (Law L5) — share a home = share its κ, and the recipient re-derives the exact desktop with no server and no sign-in. `remix(parent, edit)` derives a child manifest carrying `prov:wasDerivedFrom` the parent (ADR-0082); a home is a node in the same lineage hypergraph as any other holospace, and `walkLineage` walks it to genesis. Lineage is metadata — it does not change the visual address (content convergence: undo an edit, return to the parent κ).
4. **Create + remix with no special case (once the boot is refactored).** `csResolve`'s empty-desktop branch returns the home κ instead of scaffolding a starter; Code shows the manifest; remix edits the manifest into a child κ. Because the desktop is finally a doc, the existing window→preview FLIP lifts `#world` continuously and Done re-derives the updated home and settles it back — "the original canvas shrinks into the preview" becomes literally true, with no faked content.

**The staged remaining work (and why it is staged).** The load-bearing change is the **compositor refactor**: the shell's desktop boot must become `render(homeκ)` rather than painting native DOM, with a one-time migration that snapshots the current ad-hoc desktop into the first home manifest. This is a careful re-architecture of `#world`'s mount path (over `holo-world-rw` + `holo-render`), not a one-shot edit — done wrong it breaks the live desktop. So the foundation (the provable manifest model + the isomorphism) ships and is witnessed first; the boot refactor and the `csResolve` wiring follow as a guarded migration. Until the desktop is genuinely a pure render of its κ, the empty-desktop scaffold stands — no faked mirror.

**Honest costs.** The wallpaper is a **seed, not a snapshot** — the cosmos re-derives, never a captured frame; "exact" is *content-exact*, not frame-exact (transient animation phase is not part of the κ, by design). The manifest captures **references + layout + a defined minimal instance-state** — the round-trip set must be named per object kind (what a widget persists vs. discards), not "whatever the DOM holds." The compositor refactor is real engineering and the actual gate to the live experience. And a home κ is only as fresh as its inputs — a manifest that referenced a since-removed object κ re-derives to a missing tile, surfaced honestly, not hidden.

**Composition.** Built on Holo Cosmos (ADR-0080, the wallpaper seed), Holo Prov (ADR-0082, lineage), `holo-world-rw` (world persistence), `holo-render` (the one κ→render path), and the Create studio (`csResolve`/the FLIP); grounded in Law L1/L5 (content address + re-derivation) and the Atlas isomorphism (ADR-0043). Mints only `holo:HomeScene` over schema.org/PROV-O. Pending: the compositor refactor + `csResolve` wiring, a `#holo-home` witness row, and the os-closure re-lock.
