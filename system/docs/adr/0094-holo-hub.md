# ADR-0094: Holo Hub — a Shopify-class commerce experience for the universe of κ-addressed holo apps, **100% serverless and κ-anchored, fast in any browser**, where a PRODUCT IS a holo app addressed by its `did:holo` κ; built by COMPOSING the substrate's already-shipped commerce primitives (Own · Settle · Atlas · Scene) and **ADOPTING Medusa's commerce MODEL as a byte-pinned spec** — the "adopt the desktop, don't run it" pattern ([0029](0029-adopt-the-desktop.md)) — NOT running Medusa's server, because the substrate is server-less by law and Medusa's "serverless" means serverless-**Node**, not in-browser

**Status:** Proposed — **DESIGN ONLY**, REVISED. The first draft resolved the Node+Postgres tension by relocating Medusa off-device (governed egress). The operator made **100% serverless · κ-anchored · in-tab · fast-in-any-browser** a HARD, non-negotiable constraint, which eliminates the server rather than relocating it. This revision supersedes the off-device resolution. No app code; the feasibility note + staged plan live at [`../holo-hub-exploration.md`](../holo-hub-exploration.md); awaiting acceptance (the brief's deliverable 4).

**Context.** The ask: a *Shopify for holo-native apps* — build · publish · monetise · discover the whole universe of holo apps, every app referenced + rendered from its κ, enabling infinite nesting · interoperability · remixing · indexing · monetisation · streaming, everything streamed as **κ-addressed self-verifying objects, 100% serverless, fast in any browser**, magical, seamless, simple. Atlas ([0043](0043-holo-atlas.md)) is the verification MAP; Hub is the STORE. Three findings had to be separated before a line of design.

**FIRST — the substrate already SHIPS most of "commerce."** This is a COMPOSITION, not a platform to build:
- a **PRODUCT = a holo app = its `did:holo` κ**; the catalog (`apps/index.jsonld`, 41 datasets, each `@id=did:holo:…` with `schema:name·description·applicationCategory·image` + `dcat:landingPage`) IS a product feed, already generated by `gen-apps-catalog.mjs`.
- **RENDER from κ** = `publishSource` ([0088](0088-holo-home.md)/[0092](0092-holo-import.md)): an app is a self-contained HTML source sealed to a `schema:WebApplication` κ the substrate already renders, in-tab, 0-network.
- **INFINITE NESTING** = Holo Scene ([0089](0089-holo-scene.md)) + Holo Graph ([0046](0046-holo-graph.md)) + Holo Link ([0060](0060-holo-link.md)) — nesting = links, recursion = `verifyDeep`; a Hub is itself one κ, so a Hub nests inside a Hub.
- **DISCOVERY / INDEXING** = Atlas ([0043](0043-holo-atlas.md)) + Federate/Ask ([0038](0038-holo-federate.md)/[0041](0041-holo-ask.md)) + NANDA ([0034](0034-holo-nanda.md)).
- **OWNERSHIP** ("install"/"buy") = Holo Own ([0053](0053-holo-own.md)): a Title is a signed, content-addressed, revocable claim whose head κ proves the whole history.
- **MONETISATION** = Holo Settle ([0048](0048-holo-settle.md)) + the Plasma rail ([0068](0068-holo-settle-plasma.md), gasless USD₮ moved ONLY through Holo Wallet's default-deny signing seam) + HTTP-402 metering (`holo-api`).
- **PROVENANCE / ANALYTICS** = PROV-O receipts + Holo Prov ([0082](0082-holo-prov.md)) — analytics you re-derive, not surveillance.

The genuine GAP the substrate lacks is the commerce bookkeeping a storefront needs: collections, variants, prices per region/currency, promotions, tax, inventory, cart, checkout. **That model is exactly what Medusa is the proven, feature-complete source of truth for.**

**SECOND — "run Medusa verbatim" and "100% serverless in any browser" are mutually exclusive, and the evidence is decisive.** Medusa v2's commerce modules are built to run in **any Node environment, including serverless Node runtimes** (Lambda/edge), over **MikroORM + PostgreSQL**; SQLite is explicitly *not a core priority*, and there is **no browser/WASM execution path** (Medusa, *v1→v2 conceptual differences*; *toward Medusa 2.0*; discussions #4058, #4431). Medusa's "serverless" is **serverless-Node-with-a-database**, NOT in-browser. Running its npm packages in a tab would require Node polyfills + a SQL engine in WASM + MikroORM in the browser — unsupported and slow, the opposite of "fast in any browser." [0092](0092-holo-import.md) already classified `backend/DB → out-of-scope-v1` ("no server on a serverless device to host it"); [0029](0029-adopt-the-desktop.md) forbids importing a foreign runtime as the source of truth (Law L4). So Medusa-the-running-server cannot be the engine here — by law and by fact.

**THIRD — the OS's own deepest pattern IS the resolution: ADOPT, DON'T RUN.** [0029](0029-adopt-the-desktop.md) ("adopt the desktop, don't run it — freedesktop/KDE standards as κ-objects, the DE as a swappable skin") and [0031](0031-holo-conform.md) ("every spec is a self-verifying byte-pinned UOR object") set the template the OS uses for EVERY external standard — NANDA ([0034](0034-holo-nanda.md)), A2A ([0036](0036-holo-a2a.md)), Skills ([0035](0035-holo-skills.md)): *adopt the standard as a pinned κ-object, render natively, mint nothing*. Holo Hub adopts **Medusa's commerce MODEL** (its data model + the Store API OpenAPI contract) the same way: the model becomes the byte-pinned source of truth; the engine runs natively on the κ-substrate. This yields something STRONGER than hosted Medusa — a verifiable, κ-native, zero-server commerce engine that runs in-tab in any browser.

**The honest trade, named (pushed back on, per the brief).** This RELAXES the brief's *"run Medusa verbatim / no new code"* to *"ADOPT Medusa's model + vendor-and-compile its pure functions where they exist."* The two original constraints (run-verbatim AND 100%-serverless-in-tab) cannot both hold — Medusa is architecturally a Node+DB server. The operator chose **serverless as non-negotiable**, so verbatim-execution is what gives. This is not a compromise hack; it is the [0029](0029-adopt-the-desktop.md) pattern the whole OS is built on.

**Decision.** Holo Hub = three planes, ALL 100% serverless and κ-anchored, in-tab, fast in any browser.

1. **TRUTH plane (κ, in-tab, L5).** A **PRODUCT IS a holo app addressed by its `did:holo` κ.** The product feed is `apps/index.jsonld`. Identity, render, nesting, ownership, payment receipt, and provenance are all κ-objects re-derived in the tab. Already proven.

2. **COMMERCE-MODEL plane (Medusa ADOPTED as a byte-pinned spec — NOT run).** Pin Medusa's **data model** (entity shapes + relationships for product/variant/price/region/currency/cart/order/promotion/tax/inventory) and its **Store API OpenAPI contract** as **byte-pinned κ-objects** ([0031](0031-holo-conform.md) conform pattern), vendored from upstream at a pinned commit (the import receipt pins `git+repo@commit`, [0092](0092-holo-import.md)). This κ-object is the **source of truth** for the merchandising model and the API surface — so the Hub speaks Medusa's Store-API shape (drop-in for any Medusa-aware client/agent) without running Medusa. Where a Medusa rule is a **pure deterministic function** (price-list selection, tax calculation, promotion application, cart totals), **vendor that function and compile it to a verifiable WASM κ-transform via Holo Forge** ([0051](0051-holo-forge.md)) — maximal verbatim fidelity, still 100% serverless + re-derivable (L5). Where it is DI/ORM/HTTP service glue (not portable), realize it natively as a κ-transform over the κ-store — the unavoidable new code, exactly as [0029](0029-adopt-the-desktop.md) writes native code to honor adopted desktop standards.

3. **ENGINE + EXPERIENCE plane (the Hub holospace, a sealed κ, twin of Atlas — runs ENTIRELY in-tab).** **Persistence = the κ-store** (content-addressed objects in IndexedDB/OPFS, the `holo-store.js` Law-L3/L4 backend Holo Q already uses): every product, cart, order, and Title is a self-verifying κ-object — **no Postgres, no Node, no egress**. The storefront renders **product=app** cards from κ with Atlas's live in-tab verify (the `renderGrid` isomorphism, [0043](0043-holo-atlas.md)); Featured/collection/category/search; a one-tap **OPEN** renders the LIVE app from its κ via Scene ([0089](0089-holo-scene.md)) — infinite nesting, each leaf re-deriving. **BUY/OWN** = mint a Holo Own Title ([0053](0053-holo-own.md)) over the product κ + pay through the Wallet gate ([0068](0068-holo-settle-plasma.md) gasless USD₮ / 402), cart→checkout computed in-tab by the adopted-Medusa κ-transforms and SETTLED to a κ Title + an on-device PROV-O receipt. **PUBLISH/MONETISE** = seal an app (`publishSource`) → write a product κ-object referencing it → set price/region/promo as κ-objects against the pinned model. The Hub app is itself ONE κ — a Hub nests/remixes inside a Hub (the recursive Shopify-App-Store, every "app" a verifiable κ). The only network is the substrate's normal **governed κ-streaming of assets** ([0026](0026-sovereign-delivery.md)) — the core commerce loop is **0-network**.

**Honest costs / non-goals.**
- **This relaxes "run Medusa verbatim / zero new code."** The two original constraints are mutually exclusive (Medusa is a Node+DB server); serverless-in-tab is the one the operator made non-negotiable, so verbatim-execution gives. We ADOPT Medusa's model (byte-pinned κ) and vendor-and-compile its PURE functions ([0051](0051-holo-forge.md)); the DI/ORM/HTTP service glue is realized natively as κ-transforms — the [0029](0029-adopt-the-desktop.md) pattern, named not hidden. Claiming "Medusa runs verbatim in any browser, fast" would be the dishonesty [0092](0092-holo-import.md) refuses.
- **Fidelity is to the MODEL + API contract, byte-pinned and re-derivable — not to Medusa's process.** A Medusa-aware client/agent sees the same Store-API shapes; the engine underneath is κ-native. The empirical "which Medusa functions are extractable + Forge-compilable vs must be native" split is resolved in Stage 1, surfaced honestly (never faked).
- **The κ-store is the database.** Relational queries Medusa would push to Postgres become κ-graph traversals/indexes over content-addressed objects (the merchandising index is itself a re-derivable κ). This is real engineering, scoped per stage; durability rides the existing `holo-store.js` κ-store + the healer ([0076](0076-holo-heal.md)).
- **Listed third-party apps are UNTRUSTED** ([0092](0092-holo-import.md)/[0033](0033-holo-constitution.md)) — listing ≠ endorsement; render stays sandboxed + conscience-gated; a paid listing grants no privilege.
- **Payment finality rests on the wallet + chain anchor** ([0068](0068-holo-settle-plasma.md)), outside the substrate's re-derivation — honestly scoped, the same trust [0068](0068-holo-settle-plasma.md) states.

**Composition.** Composes Holo Atlas ([0043](0043-holo-atlas.md)), Holo Own ([0053](0053-holo-own.md)), Holo Settle + Plasma ([0048](0048-holo-settle.md)/[0068](0068-holo-settle-plasma.md)), Holo Scene/Graph/Link ([0089](0089-holo-scene.md)/[0046](0046-holo-graph.md)/[0060](0060-holo-link.md)), Holo Forge ([0051](0051-holo-forge.md), the pure-function → WASM κ-transform), Holo Import + agent-native ([0092](0092-holo-import.md)/[0093](0093-holo-import-agent.md), the vendoring + Store-API→MCP projection), Holo Conform ([0031](0031-holo-conform.md), spec-as-κ) + Adopt-the-desktop ([0029](0029-adopt-the-desktop.md), the adopt-don't-run pattern), the κ-store (`holo-store.js`) + Holo Heal ([0076](0076-holo-heal.md), durable persistence), Holo Prov ([0082](0082-holo-prov.md)), NANDA ([0034](0034-holo-nanda.md)), the conscience ([0033](0033-holo-constitution.md)). Grounded in Laws **L1** (a product IS its κ), **L3** (O(1) re-resolve), **L4** (0-network core loop; only governed κ-asset-streaming), **L5** (every product/cart/order/Title/receipt re-derives in-tab; tamper refused). **Mints nothing new** (reuses `hosc:`/`holo:`/PROV-O/schema.org/ODRL). **Witness PLAN** (owed on acceptance): `#holo-hub` + `holo-hub-witness.mjs` — Medusa model/OpenAPI pinned as a re-deriving κ · catalog → product set with κ identity · a Forge-compiled pricing/tax κ-transform re-derives byte-for-byte (L5) · product κ renders + re-derives · store-in-store round-trips one composed κ ([0089](0089-holo-scene.md)) · cart→checkout computed in-tab with 0 network · buy → a Title over the product κ + a re-derivable settle receipt ([0053](0053-holo-own.md)/[0068](0068-holo-settle-plasma.md)) · tamper (flipped product byte / forged Title) refused · baseline grants no capability.
