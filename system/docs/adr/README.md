# Architecture Decision Records

Each ADR is one decision: its context, the decision, and its consequences. They are the
durable record of *why* Hologram OS is shaped the way it is. Naming follows the
[adr-tools](https://github.com/npryce/adr-tools) convention: `NNNN-title.md`.

The numbering is shared with the engine. **ADRs 0001–0021 are engine decisions** and live
upstream in the [holospaces](https://github.com/Hologram-Technologies/holospaces) submodule
(`holospaces/docs/`). **ADRs 0022+ are product decisions** and live here.

## Writing a new ADR

1. Copy the shape of an existing ADR: **Status**, **Context**, **Decision**, **Consequences**,
   and the **external authorities** it conforms to.
2. Name it `NNNN-title.md` (next free number).
3. If it adds a conformance obligation, add its row to
   [`w3c-conformance.jsonld`](../../conformance/w3c-conformance.jsonld) and ship
   the witness with it — the decision is not done until it is witnessed (ADR-024).
