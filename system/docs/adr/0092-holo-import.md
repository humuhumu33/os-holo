# ADR-0092: Holo Import — paste a GitHub URL, get a running, content-addressed Holo app: import is a DETERMINISTIC ENCODER onto the existing publishSource/Q spine, not a new runtime, and it is HONEST about the line where "serverless" ends

**Status:** Proposed (the pure core + the static/SPA loop landed + Node-witnessed 32/32, AND wired into the live Create studio + verified live in the booted shell; the Forge bundler class and the `#holo-import` row are staged). Ships at `system/os/usr/lib/holo/holo-import.mjs` (pure, isomorphic, zero-dep, deps injected), adds one DETERMINISTIC task to the Q mux (`holo-q-mux.js`), and is wired into the shell's Create studio (`shell.html` `openCreate` binds the importer; a GitHub URL typed into the composer routes to it) — so `Q.create(repoUrl, { task: "import" })` rides the same κ-memo spine every build uses.

**Context.** The ask: in Create mode, paste a GitHub URL and have Hologram ingest it, encode it as κ-addressed objects, and run it as a first-class Holo app — 100% serverless, in-browser, low-latency, as seamless as Railway connecting a repo, except the deploy target is the **device + the κ-store**, not a cloud. Two facts had to be separated before writing a line. **First, not every repo is a browser-runnable serverless artifact.** A static site or a client SPA is; a repo with a Node/Python/Postgres backend is not, as-is — there is no server on a serverless device to host it. Pretending otherwise would ship an empty app wearing the repo's name, the exact dishonesty this OS refuses (the "never fakes" voice of [0084](0084-holo-q-mux.md)/[0083](0083-holo-q-diffusion.md)). So the load-bearing question is the **BUILD LOCUS**, decided per class, exactly as [0051](0051-holo-forge.md) (a build is a re-derivable κ-transform) and [0072](0072-holo-strategy.md) (the honest WASM-vs-CLI split) faced it. **Second — the finding that makes this SMALL** — a Holo app is already nothing but a self-contained HTML source sealed by `repo.publishSource` into a JSON-LD `schema:WebApplication` κ ([0088](0088-holo-home.md)/`holo-blocks-repo.mjs`). So *once a repo is reduced to one self-contained HTML string, it seals through the IDENTICAL envelope a studio-built app does — byte-for-byte the same κ.* Import is therefore **not a new kind of object and not a new runtime**: it is a deterministic ENCODER that produces a source the substrate already renders · links · remixes · shares · traces.

**Decision.** Make import a pure encoder on the existing spine, classified honestly per repo:

1. **Classify by CONTENT, then choose the build locus — and SURFACE the verdict.** `classify({files, pkg})` reads `package.json` + the file listing (never guesses) and returns `{class, path, runs, reason}`: **static / client-SPA → `direct`** (inlined + sealed, runs in v1); **client-bundle (vite/webpack/rollup/parcel/esbuild) → `forge`** (a re-derivable in-browser bundle, [0051](0051-holo-forge.md) — STAGED, named not faked); **SSR (next/nuxt/remix/astro) → `translate-or-forge`** (staged); **backend / DB / non-JS server / CLI / library → `out-of-scope-v1`** with the reason stated (no server to host it; the serverless line ends here). A non-runnable class returns `runnable:false` with its reason — **nothing is ever faked**.

2. **Encode the runnable class FAITHFULLY (fidelity over cleverness).** `encodeStatic` inlines each relative same-origin `<link rel=stylesheet>` and `<script src>` **verbatim** (byte-preserved, no model, no rewrite) into one self-contained HTML, preserving attributes like `type=module`. **ABSOLUTE / cross-origin / API refs are NOT inlined** — they are recorded in `externalRefs` and the result reports `selfContained:false`. This is the "log what we don't run" rule made structural: a backend call can't be served serverlessly, so we name the gap, never pretend it's covered.

3. **Seal through the EXACT publishSource envelope (the κ-parity keystone).** `sealApp({name, source, hash})` builds the same UOR object `publishSource` does (same `@context`, `@type:[SoftwareSourceCode, WebApplication]`, JCS, SHA-256), so **the imported app's κ EQUALS a studio-built app's κ for identical bytes** — proven in the witness against the real `HoloRepo`. An imported app is first-class: it renders, links, remixes, shares, and tracks lineage like any other, and drops into any holospace as one κ-object via the scene engine ([0089](0089-holo-scene.md)).

4. **Ride `Q.create` as a DETERMINISTIC task (one path for human + agent).** A new mux task `import` (marked `deterministic:true`, so discovery SKIPS it — it has no HF model, no weights, and never pretends to pick one) routes `Q.create(repoUrl, {task:"import"})` to a pure encoder specialist. It rides the κ-memo fabric ([0085](0085-holo-q-fabric.md)) exactly like every build: a repeat is **O(1) `cached:"hot"`**, and because the specialist isn't re-run on a hit, **the ingest happens ONCE — a re-import re-fetches NOTHING** (Law L3, and the precise honoring of L4: the network is an injected boundary, hit once).

5. **Mint an import receipt that walks to GitHub forever.** `importReceipt` is a PROV-O activity whose `prov:used` pins `git+<repo>@<commit>` (the SHA, resolved at the ingest boundary — a bare branch is mutable, so we address the pinned commit) and whose `prov:generated` is the app κ. It records the build-locus `class`, and names every `hosc:unservedExternalRefs` — so a remote-imported app carries honest lineage and an honest coverage statement in Holo Prov ([0082](0082-holo-prov.md)) `walkLineage`.

**Honest costs / non-goals.** **Only the static/SPA class runs in v1.** That is a real, large slice (hand-rolled SPAs, CDN-only apps, static-site output) but it is NOT "any repo" — the bundler class (`forge`) is the named, staged next step (esbuild-wasm in-browser, cached as a κ-transform), and SSR/backend/DB/CLI are classified-and-surfaced, not run. **Fidelity is byte-faithful for inlined assets but not for runtime backends** — an app that `fetch()`es its own server is encoded, but those calls reach nothing serverlessly; v1 surfaces the asset-level external refs and the ADR is explicit that translating a backend to substrate primitives (κ-store for persistence, Q for inference, `holo-gov` for governed egress, [0090](0090-holo-q-remote.md)) is separate, deeper work. **The fetch is an INGEST BOUNDARY** — `fetchRepo` is injected (the browser fills it with a `holo-gov`-brokered fetch to raw.github / a κ-mirror / IPFS; the witness mocks it); the module itself never fetches and never hashes (Law L2/L4). **Imported code is UNTRUSTED** — it stays in the app sandbox and under the conscience ([0033](0033-holo-constitution.md)); arbitrary-GitHub-code is the real attack surface and is treated as such (the studio adopts the sealed source the same way it adopts any pasted source — no new privilege). **The two κs of [0091](0091-holo-q.md) still hold**: the fabric compute κ (the encoder's output value) and the published app κ (`publishSource` over `{name,text}`) are distinct correct layers; the user sees the one published κ.

**Composition.** Composes [0091](0091-holo-q.md) (rides `Q.create` as task `import`), [0051](0051-holo-forge.md) (the staged bundler κ-transform), [0089](0089-holo-scene.md) (the imported app is one scene κ-object), [0082](0082-holo-prov.md) (the receipt walks to GitHub), [0090](0090-holo-q-remote.md)/`holo-gov.js` (the governed ingest fetch), [0080](0080-holo-cosmos.md) (κ-streamed heavy assets, the LOD follow-on), and seals through `holo-blocks-repo.mjs` `publishSource` (κ-parity). Grounded in Laws L1 (the app IS its κ, named by content not URL), L3 (O(1) re-import, ingest-once), L4 (the network is a boundary, the loop is 0-network once imported), L5 (same repo@commit → same app κ; a flipped byte → a different κ, refused). Mints only `hosc:Import` over PROV-O/schema.org. Witnessed `system/tools/holo-import-witness.mjs` 32/32 (emits `holo-import-witness.result.json`, EARL `earl:TestResult`) against the REAL `holo-blocks-repo` + `holo-scene` + Q/trinity/mux: parse (incl. bare `github.com/owner/repo`) · classify the four classes truthfully · byte-faithful inline · external refs surfaced · **imported app κ === publishSource κ** · re-import re-derives the same κ · tamper refused · receipt pins git+commit and names unserved refs · scene round-trip · **`Q.create({task:import})` O(1) hot + re-fetches nothing** · backend/bundle return `runnable:false` with the reason. VERIFIED LIVE in the booted shell (the FHS gateway): the module loads in-browser and is fully isomorphic (parse · classify · byte-faithful inline · **κ-parity `sealApp.id === publishSource.id` holds in the real browser**); `openCreate` binds the importer (`routeTask("import") → holo-import`); and `window.Q.create("github.com/octocat/Spoon-Knife", {task:"import"})` ingests the REAL repo (commit pinned), classifies runnable, encodes faithfully, seals a real app κ, and a **repeat is O(1) `cached:"hot"` (~0.06 ms) at the same κ — the ingest happens once**; the `send()` route-predicate sends GitHub URLs to import and every natural-language prompt to build.
